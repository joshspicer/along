# Contributing

Thank you for contributing to Along.

Before opening a change:

1. Discuss substantial product or architecture changes in an issue first.
2. Keep changes focused and include tests for behavior changes.
3. Run the relevant Go and Flutter checks described in the repository workflows.
4. Never include credentials, real user content, invitation links, diagnostic
   reports, production logs, or generated secret files.

By submitting a contribution, you agree that it is licensed under the GNU
Affero General Public License version 3, as described in [LICENSE](LICENSE).

Report security vulnerabilities privately through GitHub Security Advisories,
following [docs/SECURITY.md](docs/SECURITY.md). Do not open a public issue for
credentials, real user data, or an unpatched vulnerability.
